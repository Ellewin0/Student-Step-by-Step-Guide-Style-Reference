Student Step-by-Step Guide Style Reference

This page defines a reusable visual style, tone, and structure for creating student-facing guides about assistive technology, software, learning tools, and support processes.

Use this page as a style reference only. It does not contain organisation-specific procedures, internal policy, private information, or tool-specific instructions.

1.0 Purpose of this style reference

1. What is it?

This style reference helps an agent create consistent student-facing guides from process notes, rough instructions, or tool information.

The agent should use this page to understand the preferred document style, not as a source of procedural content.

Important: This page is a formatting and style reference only. It must not be treated as a source of support rules, eligibility criteria, service commitments, or technical procedures.

2.0 Visual style rules

1. Overall look and feel

The style should feel clear, friendly, modern, practical, and easy to scan. It should support students who may be learning a tool for the first time or returning to instructions while completing a task.

Element Preferred style How to use it
Font Arial, Helvetica, or another clear sans-serif font Use consistent font styling across title, headings, body text, tables, and callouts.
Page background Warm off-white or soft sand background Use a calm background that makes content feel less dense.
Cards and sections White cards with dark rounded borders Group content into clear blocks so students can focus on one part at a time.
Borders Dark, rounded, slightly bold borders Use borders to separate sections, tables, tips, screenshots, and key reminders.
Colours Soft blue, green, yellow, purple, coral, and grey Use colour gently for scanning and emphasis. Do not rely on colour alone to communicate meaning.
Spacing Generous spacing between sections Avoid cramped text. Use short paragraphs and separate steps clearly.

2. Colour palette

Use soft colours to organise information, reduce visual load, and make guides feel approachable.

Soft blue
Use for notes or general information.
Soft green
Use for examples, success states, or helpful options.
Soft yellow
Use for tips and key reminders.
Soft purple
Use for table headers or section highlights.
Soft coral
Use sparingly for cautions or items requiring confirmation.
Soft grey
Use for neutral navigation blocks or placeholders.

3.0 Language and tone

1. Tone

Use a casual-professional tone that is supportive, calm, and confidence-building. The guide should help students feel capable, not overwhelmed.

Use

  • Plain English
  • Short steps
  • Supportive wording
  • Clear action verbs
  • Practical tips

Avoid

  • Jargon without explanation
  • Long dense paragraphs
  • Overpromising support
  • Unconfirmed contact details
  • Internal-only language

2. Writing style

Write directly to the student using clear instructions. Use phrases such as “Select”, “Open”, “Check”, “Choose”, “Type”, “Save”, and “Review”.

Tip: If a step might be difficult to follow without a visual, add an image placeholder and an accessible description.

4.0 Default guide structure

1. Recommended section order

Use this structure when creating a detailed student guide.

  1. Title page: Include the guide title and audience, such as “Student Guide”.
  2. Contents: List the main numbered sections in order.
  3. Introduction: Explain what the guide is for and what the student will learn.
  4. What is it? Explain the tool, process, or support option in simple language.
  5. Getting started: List what the student needs before they begin.
  6. Step-by-step instructions: Break the process into short, sequential steps.
  7. Features or options: Use a table if students need to compare choices.
  8. Troubleshooting: Include common issues and simple next steps.
  9. Frequently asked questions: Add likely student questions and short answers.
  10. Where to get help: Include confirmed support options or placeholders.

5.0 Reusable components

1. Tip callout

Tip: Keep tips short and practical. Use them to reduce confusion, prevent common mistakes, or help students prepare.

2. Note callout

Note: Use notes for neutral reminders, background information, or helpful context.

3. Confirmation or caution callout

Confirm: Use caution-style callouts when information needs checking before publication.

6. FAQ format

6.0 Accessibility rules

  • Use clear headings in a logical order.
  • Use descriptive link text. Avoid “click here”.
  • Provide image descriptions for all meaningful images.
  • Do not rely on colour alone to communicate meaning.
  • Keep tables simple and use clear column headings.
  • Use short paragraphs and enough spacing between sections.
  • Use plain English and explain technical terms.
  • Mark missing or unconfirmed information with clear placeholders.

7.0 Agent instructions

Use the following instruction block inside an agent that creates student-facing guides.

Use this public style page as a formatting, visual style, tone, and structure reference only.

Do not treat this page as a source of procedural content, service rules, policy, eligibility criteria, contact details, or technical instructions.

When creating a guide, use only:
- the process information provided by the user,
- approved knowledge sources connected to the agent, or
- clearly marked placeholders where information is missing.

Follow the page's preferred style:
- clear title and guide type,
- contents section,
- numbered major sections,
- short subheadings,
- plain English explanations,
- step-by-step instructions,
- practical Tip callouts,
- simple tables,
- screenshot placeholders,
- accessible image descriptions,
- troubleshooting section,
- frequently asked questions,
- where to get help section.

Match the visual feel as closely as possible:
- clear sans-serif font,
- warm off-white page background,
- white content cards,
- dark rounded borders,
- soft colour highlights,
- generous spacing,
- easy-to-scan layout.

Do not invent missing process steps, support contacts, policy details, access rules, or service commitments. If information is missing, insert a clear placeholder and mark it for confirmation.

8.0 Review checklist

  • The guide uses this page as a style reference only.
  • The content is based on the user-provided process or approved sources.
  • No organisation-specific details have been added unless provided by the user.
  • Missing information is marked with placeholders.
  • The tone is clear, practical, supportive, and student-friendly.
  • The guide includes step-by-step instructions where appropriate.
  • Images or screenshots have accessible descriptions.
  • The final guide is easy to scan on desktop and mobile.

This page is designed to be published as a shallow public website knowledge source for an agent. It contains style guidance only and intentionally excludes internal procedures, private information, and organisation-specific references.
